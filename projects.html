Projects

Android App Development

Various apps published on the Play Store

Over the course of a couple of years I have published Android Applications to the Google Play Store. Majority of these are utility applications created simply to assist users by adding convenience to their daily lives. Applications are written in Java with XML for the UI and utilize SQLite as the DBMS.

Simple Weather - Under Development
  • A Simple Weather Application for Android. Written primarily in Java, however there will be some Kotlin classes as well.
  • The primary objective of this project is so I can learn to utilize Google's Snapshot API in a real world application. This is a simple weather application which will use the Snapshot API to get the weather information at the user's current location. Initially I plan to use this project to get familiar working with the API, however once this is complete, I plan to expand upon this project with extra functionality.
Smart Actions - Under Development
  • Allows the user to set up triggers which (when fired) perform a set of actions. For example, when the user is at work, the Region Trigger would active and put the phone into silent mode (VolumeAction), turn down the music volume (VolumeAction), turn off bluetooth (BluetoothAction) and turn down the brightness (DisplayAction).
  • The implementation of triggers and actions is done in a modular way so that more can be added easily. This API can eventually be opened up to developers, allowing them to create their own triggers and actions.
  • Poweruser features will be added as well. An NFC Trigger is planned so the user can have actions occur just by tapping the device to a NFC tag. And a REST Action is also planned so when the user gets to work, they can have a RESTful Call automatically fire out to their server at home updating their location to work. Or if they have a home automation suite which can consume RESTful calls, they can have the lights turn on once they arrive home.
RiLo (Ring Louder)
  • Allows the phone to ring, even if on silent, when a specified person calls for the nth time. This is useful for ensuring an important call (spouse, parent, child, manager, etc.) is not missed. This feature can also be used to locate a misplaced phone which is on silent mode.
Seer (Silence Everything Except Ringer)
  • Seer fixes a design issue which was introduced in Android 4.0, which is not being able to have seperate volume levels for ringer and notifications.
Auto Off
  • Auto Off helps the user conserve battery by disabling Wifi/Bluetooth when not in use. The user sets up a specified timeout, once the device is disconnected from the network and the timeout period occurs, the wifi radio is disabled. This functions similarly for bluetooth devices.
Power Wifi
  • Power Wifi allows for actions to be taken based off the presence of power. For example when you sleep at night, you can plug in the phone and have the ringer silenced and the alarm volume raised. This adds convenience to a users life by taking care of tasks which are normally preformed manually.

IoT Multi-Sensor

Sensor which posts data to a server

Using an ESP8266 microchip, I created a custom Internet of Things Multi-Sensor.This sensor is capable of detecting motion, temperature, humidity, luminance, and carbon monoxide. The chip connects to the network via Wi-Fi and uses MQTT to post to a MQTT Server running on my desktop. Some effeciency measures are also implemented, such as polling the sensors after certain intervals. The programming is done primarily in C++ using Arduino IDE.

Home Automation

Using a python based H.A. server

Currently using an open source python based home automation server to automate my home. Home Assistant is a wonderful tool to get started with bridging the gaps between current day home automation hubs/devices. I have configured our setup at home to meet the needs of the family; this consists of monitoring of temperature, motion, state of devices in order to perform the appropriate action, such as sending a notification. This setup even allows control via Google Home and Amazon Echo. I have utilized linux to stand up the server and python to configure this platform.

Crypto Tracker

Crypto currency price monitoring device

Using an esp8266 microprocesser at its core, I created a crypto currency price monitoring tool with a screen. This device utilizes the cryptocompare api via webservice calls. The user can configure which crypto they'd like to track, the frequency in which to reload, as well as which fiat currency to compare to.
